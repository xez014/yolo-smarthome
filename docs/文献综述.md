# 基于YOLO的智能家居视觉感知与历史物品检索系统

# 文献综述

## 引言

随着物联网（Internet of Things, IoT）技术的不断成熟以及智能家居生态的日益完善，家庭视觉感知已成为人工智能在民用领域落地的重要方向之一。传统的家庭监控系统仅提供被动式的视频录像与事后人工回放功能，缺乏对画面语义内容的主动理解能力。如何利用深度学习驱动的目标检测与多目标跟踪技术，将视频帧中的物品实时转化为结构化的时空数据，并通过 Web 数据库实现"物品何时在何处出现过"的智能检索，是当前智能家居领域亟待解决的关键问题。

本文围绕"基于YOLO的智能家居视觉感知与历史物品检索系统"这一课题，从**目标检测算法的演进与优化**、**多目标跟踪算法的研究进展**、**智能家居场景下的视觉应用**以及**云边协同流媒体系统架构**四个维度，对国内外相关文献进行系统性的梳理与评述，为本课题的技术选型与系统设计提供理论依据。

## 1 目标检测算法的演进与优化

### 1.1 YOLO系列算法的发展脉络

目标检测（Object Detection）是计算机视觉的基础任务之一，其核心目标是在图像中定位并识别感兴趣的目标类别。传统的两阶段检测器（如 R-CNN 系列）虽然精度较高，但推理速度难以满足实时性需求。2016 年，Redmon 等人提出了 YOLO（You Only Look Once）算法，首次将目标检测建模为端到端的单阶段回归问题，通过将输入图像划分为网格并在单次前向传播中同时预测边界框坐标与类别概率，极大地提升了检测速度，开创了实时目标检测的新范式[8]。

此后，YOLO 系列经历了多次关键的架构演进。YOLOv2 引入了锚框（Anchor Box）机制与批归一化（Batch Normalization），改善了定位精度与召回率。YOLOv3 采用 Darknet-53 骨干网络和多尺度预测策略，增强了对不同尺寸目标的检测能力。YOLOv4 进一步融合了 CSPDarknet53 骨干、Mosaic 数据增强等训练技巧，在精度与速度之间取得了更优的平衡。YOLOv5 转向了更轻量化、易于部署的工程化路线。在最新的迭代中，**YOLOv8 采用了无锚点（Anchor-free）检测头与改进的 C2f（Cross-Stage Feature-fusion）模块**，简化了损失函数设计，同时实现了检测、分割、姿态估计等多任务统一框架[9]。而 2024 年发布的 **YOLOv11 在此基础上将 C2f 升级为 C3k2 模块，并引入了空间注意力机制**，在参数量减少约 22% 的前提下进一步提升了推理的平均精度（mAP），使其在边缘计算设备上具有更高的能效比[12]。

### 1.2 面向室内场景的轻量化改进

室内环境相较于室外场景具有其独特的复杂性：目标遮挡频繁、光照条件多变、小目标（如手机、钥匙、遥控器等）占比高。针对上述挑战，国内外学者围绕 YOLO 开展了大量面向室内场景的改进工作。

张华等[1]针对智能家居设备算力有限的特点，提出了一种面向智能家居的轻量型 YOLO 目标检测网络设计方案，通过引入轻量化骨干网络替换原始的 Darknet 结构，并优化特征金字塔（FPN）的通道数分配，在树莓派等嵌入式平台上实现了可接受精度下的实时推理。刘洋等[3]专门研究了复杂光照条件下基于注意力机制的室内小目标检测算法，通过在 YOLO 的 Neck 部分嵌入 ECA（Efficient Channel Attention）通道注意力模块，使网络自适应地聚焦于关键特征通道，有效缓解了室内光照突变和阴影遮蔽对小物品识别的干扰。赵雷等[5]提出了结合轻量化特征金字塔的室内监控视频物体快速识别方法，通过引入深度可分离卷积（Depthwise Separable Convolution）重构 PAN（Path Aggregation Network）的特征融合路径，在不显著损失精度的情况下将 Neck 部分的计算量压缩约 40%。

在国际研究中，Lee 等[9]提出了面向智能室内零售环境的增强型 YOLOv8n 算法，通过设计新型 C2f-ParNet 结构并嵌入 CARAFE（Content-Aware ReAssembly of Features）上采样模块，针对性地改善了复杂室内背景下的特征提取质量，在自建的室内购物数据集上取得了优于基线模型的检测精度。Garcia 等[11]则探索了 YOLO 在智能建筑全景图像中的应用，对比了 YOLOv5 与 YOLOv8 在变形球面图像中检测室内物体的有效性，实验表明 YOLOv8 在处理宽视角畸变图像时表现出更强的鲁棒性。

上述研究表明，**通过注意力机制嵌入、轻量化结构替换和特征融合路径优化等手段改进 YOLO 架构**，能够有效提升其在室内复杂环境中的适用性，这为本课题选用 YOLOv11 作为核心检测模型并进行室内场景微调提供了充分的理论依据。

## 2 多目标跟踪算法的研究进展

在实时视频监控中，仅完成单帧目标检测是不够的，还需要跨帧维护每个目标的身份连续性，即多目标跟踪（Multi-Object Tracking, MOT）。MOT 的经典范式为"检测后跟踪"（Tracking-by-Detection），其中跟踪器利用检测器输出的边界框，通过运动预测与数据关联算法建立帧间目标的对应关系。

SORT（Simple Online and Realtime Tracking）算法通过卡尔曼滤波器（Kalman Filter）进行目标运动预测，再利用匈牙利算法（Hungarian Algorithm）进行 IoU（Intersection over Union）距离的最优匹配。DeepSORT 在此基础上引入了基于 ReID（Re-Identification）特征的深度外观描述符，增强了目标在遮挡后重新出现时的重识别能力。但 DeepSORT 的 ReID 模块引入了额外的计算开销，不利于实时部署。

李四[4]研究了边缘计算环境下 YOLO 与 DeepSORT 联合应用的室内安防实时追踪模型，指出 DeepSORT 虽然在跟踪稳定性上优于 SORT，但其 ReID 分支导致推理延迟增加了约 30%-50%，在嵌入式设备上难以达到实时要求。

Zhang 等人于 ECCV 2022 提出的 **ByteTrack** 算法则另辟蹊径：传统跟踪器通常仅关联高置信度的检测框，而丢弃低置信度框。ByteTrack 的核心创新在于**将几乎所有低置信度的检测框（"every detection box"）也纳入关联过程**，通过二级级联匹配策略，先关联高置信度检测，再利用低置信度检测恢复因遮挡或运动模糊而暂时丢失的轨迹。这一设计显著减少了轨迹碎片化（Fragmentation）和漏检（Missing Detection），同时由于不需要额外的 ReID 特征提取网络，保持了极高的计算效率。Wang 等[12]在复杂室内场景的研究中也证实，ByteTrack 与轻量级 YOLO 检测器的组合能够在保持 30 FPS 以上帧率的同时，有效降低因遮挡导致的 ID 切换（ID Switch）率。

综上，**ByteTrack 以其轻量、高效且不依赖 ReID 的特性**，成为本课题实现家居物品持续跟踪的最佳选择。但需要注意的是，ByteTrack 本身仅负责帧间 ID 的连续性维护，对于因摄像头噪点、光照闪烁导致的"目标闪现"问题，仍需在应用层设计额外的轨迹稳定性过滤机制（如设定最低连续追踪帧数阈值后方可持久化入库），以保证数据库中存储的物品记录具有可靠性。

## 3 智能家居场景下的视觉应用

### 3.1 室内安防与行为分析

目标检测技术在智能家居中最广泛的应用方向之一是室内安防与行为分析。王晓明等[2]基于改进的 YOLOv8 模型研究了室内跌倒及异常状态检测算法，通过对数据增强策略和细化特征图的优化，实现了独居老人在复杂家居环境中跌倒行为的高精度识别。Smith 等[8]提出了一种 YOLO 自适应家庭自动化系统，将目标检测（检测人体存在与姿态）与环境传感器数据融合，用于动态调控家电运行状态以提升能效和用户舒适度。陈涛等[6]将改进的 YOLO 目标检测机器视觉技术应用于智能家居中的常见物品识别，验证了迁移学习策略对解决小规模领域数据集不足问题的有效性。

### 3.2 物品检索与空间记忆

相比上述以"安全防护"为导向的应用，将物品检测结果结构化地存储至数据库、并提供"何时何地出现过某物品"的检索服务，是一个相对新颖且实用需求强烈的方向。Chen 等[10]在面向室内导航的研究中设计了双 YOLO-SSD 混合网络，虽然其直接目标是辅助视障人群的室内行走，但其将检测结果与空间坐标映射并持久化的技术思路，为物品空间检索提供了参考。然而，现有研究中**将目标检测与数据库检索深度结合、实现"家居物品数字化资产管理及历史轨迹寻回"的完整系统方案仍然较为缺乏**，这正是本课题的核心创新点所在。

## 4 云边协同流媒体系统架构

### 4.1 传统拉流与 RTSP 协议

在标准的视频监控架构中，摄像头充当 RTSP（Real Time Streaming Protocol）服务端，后端推理服务器作为客户端通过 OpenCV 等工具主动拉取（Pull）视频流进行处理。这种"拉流"模式要求推理服务器与摄像头处于同一局域网或可通过公网直连，在云服务器部署场景下，当摄像头位于用户家庭内网而推理服务部署于远端云端时，网络穿透与端口映射成为棘手的工程障碍。

### 4.2 基于浏览器的客户端推流架构

为突破上述限制，郭健等[7]提出了基于 WebRTC 的云边端协同视频检索与实时检测系统设计方案，利用现代浏览器内置的 `getUserMedia` API 在客户端直接采集摄像头画面，通过 WebSocket 或 WebRTC DataChannel 将视频帧以二进制数据推送（Push）至云端进行推理。这种"客户端推流"模式彻底消除了对摄像头直连与内网穿透的依赖，使得任何拥有浏览器和摄像头的终端设备均可成为视频源。

然而需要注意的是：第一，现代浏览器的安全策略要求 `getUserMedia` 调用必须在 HTTPS 协议下运行，这对部署的 SSL 证书配置提出了硬性要求[7]。第二，相比原生 RTSP 拉流的高帧率（≥25 FPS），浏览器推流受限于 JavaScript 执行效率与网络上行带宽，通常需要在前端进行帧率控制（如 5-10 FPS）以平衡延迟与服务器负载。第三，推流模式下的检测为无状态推理（每帧独立、无跨帧 Track ID），若需将识别结果持久化至数据库，必须在应用层设计额外的聚合去重与节流写入策略，以避免同类物品在同一帧中产生重复记录导致数据库唯一约束冲突。

### 4.3 容器化部署与反向代理

在系统工程层面，Docker 容器化技术已成为深度学习应用部署的事实标准。通过编写 Dockerfile 与 docker-compose 编排文件，可以将前端 Nginx 静态服务、后端 FastAPI 推理服务与 MySQL 数据库封装为相互隔离又可通信的服务单元，实现"一键式"的环境复现与弹性扩展。同时，Nginx 反向代理在此架构中承担了前后端同域化、WebSocket 协议升级转发（通过设置 `Upgrade` 和 `Connection` 头部）以及跨域请求（CORS）统一管理等关键职责。

## 5 文献评述与总结

通过对上述文献的系统梳理，可以得出以下结论：

**(1)** 在目标检测算法层面，YOLO 系列经过十余年的持续演进，已从最初的概念验证发展为包含检测、分割、跟踪等能力的统一框架。最新的 YOLOv8/YOLOv11 在推理速度、参数效率和多尺度检测精度上均达到了当前最优水平，且 Ultralytics 生态提供了成熟的训练、验证与部署工具链，为本课题的模型层提供了坚实的技术底座[1][9][12]。

**(2)** 在多目标跟踪层面，ByteTrack 以其"关联一切检测框"的设计理念和无需 ReID 的轻量化特性，在实时性与跟踪准确性之间取得了出色的平衡，适合作为本课题中物品轨迹稳定性过滤的核心组件[4][12]。

**(3)** 在智能家居应用层面，现有研究多聚焦于安防行为分析（跌倒检测、入侵告警）和静态场景识别，**将目标检测深度结合结构化数据库、实现家居物品的时空维度检索与"视觉记忆"能力的全栈系统**，在公开文献中仍属空白[2][6][8]。本课题正是填补这一空白的尝试。

**(4)** 在系统架构层面，"客户端推流 + 云端推理"的 B/S 架构模式突破了传统 RTSP 拉流对网络拓扑的依赖，为跨公网远端监控提供了创新的解决方案，但同时也带来了帧率控制、无状态推理数据持久化以及 HTTPS 强制要求等新的工程挑战[7]。

综上所述，本课题在现有 YOLO 目标检测、ByteTrack 多目标跟踪以及 WebSocket 流媒体传输等技术基础上，创新性地构建"检测-跟踪-聚合-入库-检索"的全链路系统，具有明确的理论依据和工程可行性。
