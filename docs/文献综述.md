# 文献综述

## 1. 引言

随着物联网技术的发展，现在很多家庭都安装了智能摄像头。不过目前的家庭监控绝大多数都只能被动录像，要找点什么东西只能自己去拖进度条，摄像头本身并不知道画面里有什么。很多时候如果在家找不到手机或者钥匙，想回头去翻监控其实非常麻烦。要解决这个问题，就需要用目标检测和目标跟踪技术，把实时视频里出现的物品坐标、类别提取出来存进数据库里，这样就能做一个可以搜索的“家庭物品历史轨迹记录系统”。

为了给本项目的开发和算法选型提供一些参考依据，我重点查阅了目标检测算法的发展、多目标跟踪技术的应用，以及智能家庭环境下的视觉检索系统相关文献，并进行了整理。

## 2. 目标检测算法的演进与轻量化

目标检测主要是为了在图片里找出感兴趣的物体并且标注出位置。早期算法速度很慢，不太可能用在实时监控里。自从 2016 年 YOLO（You Only Look Once）算法发布后，它把检测做成了一个单阶段网络，虽然精度上有妥协，但速度有了质的飞跃[8]。

这几年 YOLO 系列发展特别快。从早期的 YOLOv3 到现在，模型结构经常在变化。比如这几年大家用的比较多的 YOLOv8 放弃了传统的 Anchor 框，而且也把损失函数简化了，用起来很方便[9]。到了最近出的 YOLOv11，它在原本 C2f 模块的基础上做了升级，还加了一些注意力机制，作者的目的是为了在减少参数的同时提高小目标的识别率，并且非常强调它在边缘设备上的能效比[12]。

另一方面，室内的环境比起室外不仅光照比较复杂，而且家里零碎的小物件（像杯子、遥控器）很多。不少学者的研究重点就是怎么把 YOLO 改造得更适合室内并且占用更少资源。张华等人[1]就针对智能家居端侧算力比较弱的痛点，给 YOLO 换了更轻量的骨干网络来调整特征金字塔通道，让模型在树莓派上也能跑出不错的帧率。刘洋等[3]研究了复杂光线下的室内小目标检测，他们在网络里加了 ECA 注意力模块，这样遇到有阴影或者局部灯光暗的时候，模型也不容易把小件物品漏掉。另外赵雷等[5]也通过深度可分离卷积重构了特征融合的路径，主要目的还是为了压缩计算量。

国外的研究也很多，比如 Lee 他们[9]针对室内零售场景优化了 YOLOv8n，用自定义的模块改善背景太乱时候的特征提取。Garcia 的团队[11]还对比了不同版本 YOLO 在宽视角畸变摄像头下的表现。总体来看，要想在普通硬件上做室内目标检测，目前比较成熟的做法就是选一个轻量版的模型（比如 YOLOv8 或 v11 的 n/s 版本），然后加入注意力机制或者做结构剪枝。 

## 3. 室内多目标跟踪的技术路线

在做视频监控的时候，如果只是单纯做目标检测，屏幕上的框很容易一闪一闪的跳，而且系统也不知道上一秒的杯子是不是这一秒的这一个。这就需要用到多目标跟踪（MOT）算法。现在大部分用的是“检测后跟踪”的思路，也就是先拿 YOLO 框出来，再想办法把帧与帧之间的框连起来。

以前比较多的是用 SORT 算法算一下 IoU 匹配，或者像 DeepSORT 那样加一个 ReID 特征提取[4]。但李四在他的论文[4]里也提到，如果把 YOLO 和 DeepSORT 放在小设备上跑，DeepSORT 提取特征那个部分会导致整体卡顿，延迟会明显上升。

所以最近一两年大家更倾向于用一种不需要额外提取特征的纯运动学跟踪算法，其中 ECCV 2022 年出来的 ByteTrack 算法讨论度很高。以往的算法往往直接把置信度比较低的那个检测框扔掉，但 ByteTrack 的思路是把那些分数偏低的框也先留着，用多级匹配的方式尽量把被遮挡住的物体轨迹接上。Wang 等人[12]的研究发现，在家里这种走动经常会遮住背后物品的场景里，用这种方法能减少 ID 切换（也就是同一个物品突然换了个新编号）的情况，而且因为它不用算 ReID 排场，基本不耗费什么额外的硬件资源。

## 4. 智能家居视觉系统及流媒体架构现状

目前目标检测用在智能家居里，绝大部分是为了安全防范。比如王晓明等人[2]研究的主要是怎么用 YOLOv8 看老人在家里是不是摔倒了。或者是像 Smith 等人[8]做的系统，通过检测人到底在不在客厅，来自动调节灯光。陈涛等[6]测试了把家用常见物品的数据做迁移学习，看识别率有没有提升。

但如果是像我在本系统里要做的——不光是检测出物品，还要把它和空间坐标绑定，变成一条条数据存进数据库搞一个“历史检索系统”的，相关的完整文献还比较少见。关于把物品坐标存下来，Chen 等[10]在做视障人士室内导航导盲的时候有一些相关的做法可供参考，但他们的应用场景不是家庭资产记录。

最后，在系统架构层面，常规的检测其实是服务器直接去拉局域网里的 RTSP 视频流。但如果不和摄像头在同一个局域网（比如部署在云端），穿透内网特别麻烦。郭健的团队[7]介绍了一种基于 WebRTC 和 WebSocket 的协同思路，思路是浏览器获取本地摄像头画面，然后不断推压视频流数据到后端的云服务器。这是一种比较巧妙的解决跨网视频传输的办法。不过在工程实现上需要自己去处理双向通信的断线情况，还要控制一下前后端推流时的帧率和数据量。另外为了更好的落地部署，把这种前后端加上数据库的代码用 Docker 跑容器化也是比较顺应现在开发习惯的一种做法。 

## 5. 总结

结合以上查阅的文献，可以看出目前基于深度学习的家庭视觉技术整体正在从简单识别走向具体的行为分析和物联网互动。YOLO 的不断升级解决了速度和精度平衡的问题[1][9][12]，ByteTrack 在不需要大算力的前提下有效解决了物品框闪跳和丢失的毛病[4][12]。

但这方面大多数系统还停留在监控和行为报警上，专门把物品坐标作为空间数据存进关系数据库供用户事后进行“物品地点溯源”的探索还比较少。所以我这个课题参考这些成熟的网络结构和工程架构规范，尝试把目标检测模型部署在服务端，通过浏览器 WebSocket 推流的轻量形式，打造一个实用的家庭物品检索记录系统，整体的技术路线是有据可查且具备可行性的。
