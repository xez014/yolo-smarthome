# 毕业设计（论文）任务书
**课题名称**：基于YOLO的智能家居视觉感知与历史物品检索系统

## 1. 毕业设计（论文）要求及原始数据（资料）
### 1.1 系统开发及性能要求
1. **可靠性与稳定性**：系统需要具备 7x24 小时处理视频流的能力。系统必须通过 Python 多线程或多进程架构解耦图像帧读取与模型推理，确保不因推流堵塞导致内存溢出或卡顿。
2. **实时推理性能要求**：依托服务端（或边缘端）硬件设备加速，对于室内 20 类典型家居物品的目标检测精度 (mAP@0.5) 应在可接受范围内，且单路视频流推理速度保证在 25-30 FPS 及以上。
3. **架构与协同要求**：应用要求具备纯 B/S 访问架构。除传统支持拉取 RTSP/本地视频 外，必须实现创新的“客户端摄像头采集 -> WebSocket 传输 -> 云端 YOLO 推理 -> 画框并回传 Web 渲染”能力。
4. **数据库与并发要求**：对检测到的连续重复物品，应当引入合适的缓存聚合机制过滤瞬间的多余数据后再入库（MySQL），避免大量请求触发锁表。

### 1.2 原始数据（资料）与开发环境
- **公开数据集**：系统模型训练的原始资料基于开源的 **MS COCO 2017 数据集**，需根据家庭场景重新编写 Python Pipeline 解包提取，过滤并汇编为专属的 `SmartHome-COCO` 标注数据再投入训练。
- **软件开发环境**：
  - **操作系统**：Windows 11 / Debian 12
  - **核心计算框架**：Python 3.10+ / PyTorch 2.0+ / Ultralytics
  - **后端服务器框架**：FastAPI / Uvicorn / SQLAlchemy
  - **前端与可视化**：Node.js 18+ / Vue 3 / Vite / Element Plus / ECharts
  - **持久化及部署**：MySQL 8.0 / Docker / Docker Compose

## 2. 毕业设计（论文）主要内容
本课题旨在设计与搭建一套具有全栈特征的智能家居视觉大屏管理系统，具体包含如下四大部分的主体内容：

1. **定制化目标检测模型的打磨与训练**
   利用 Ultralytics 开发框架及最新的 YOLO 轻量级结构权重（针对性优化 C2f/C3k2 层及 FPN 金字塔的注意力感知计算），通过数据增强、学习率衰减等策略训练模型，达成针对杯子、刀叉、盆栽、沙发、人体等高频家具物品的首帧级高精识别与抗遮挡。
2. **算法工程与后处理技术**
   在后端引擎中接入 ByteTrack 等实时多目标跟踪策略，结合内存缓存聚类逻辑，精确分配 TrackID，以有效剔除画面变化或轻微遮挡引起的物品检测闪跳和重复录入，从而提取高价值的“稳定存在物品”特征及截帧快照入库。
3. **“推拉双模”的数据流互通与处理**
   研究实现多协议视频源支持技术：支持服务器端利用 OpenCV 直连 RTSP 并 MJPEG 分发拉流；同时，为解决跨公网远程监控设备的穿透接入难题，基于 HTML5 `getUserMedia` 技术在本地浏览器端捕获实时摄像头，通过 WebSocket API 进行高频双工通信推演画面，完成云端无状态推理。
4. **前端看板与数据链路打通**
   采用最新的 Vue 3 渲染引擎和生态，搭建包括自动化系统部署向导（Setup）、实时统计展示看板（Dashboard）、分类历史截帧快照检索（History）等于一体的前端交互系统，使普通用户能如检索文本般搜索“某物品昨天在客厅哪个位置”。
