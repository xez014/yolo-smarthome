# 基于YOLO的智能家居视觉感知与历史物品检索系统 —— 开题报告

## 1. 选题意义
随着物联网(IoT)与智能家居技术的快速普及，现代家庭普遍配备了智能摄像头等视觉采集设备。然而，传统的智能家居监控多以“被动记录”与“事后回放”为主，缺乏对画面内容的深度理解与主动感知能力。在快节奏的现代生活中，用户经常遭遇“物品遗忘或遗失”（如手机、钥匙、水杯不知道放在了哪里）的痛点。

本课题《基于YOLO的智能家居视觉感知与历史物品检索系统》旨在赋予智能家居环境“视觉记忆”能力。通过引入先进的计算机视觉技术（轻量化 YOLO 系列算法与 ByteTrack 多目标跟踪算法），将实时视频流中的家居物品转化为结构化的时空数据。该研究不仅能大幅提升监控画面的语义分析精度，也能解决普通家庭的“寻物难题”，真正发挥智能摄像头的价值；同时，研究如何将深度学习模型轻量化并以 C/S（或 B/S 推流）架构融合到 Web 端，具有极强的工程应用价值和商业落地潜力。

## 2. 国内外研究现状概述
近年来，目标检测算法与智能分析系统在学术界与工业界发展迅速：
- **在目标检测算法方面**，自从 YOLO（You Only Look Once）系列问世以来，实时检测的精度和速度不断突破。2023-2024年推出的 YOLOv8 及最新的 YOLOv11 优化了 C2f/C3k2 网络结构及特征金字塔（FPN），大幅提升了对室内小目标、遮挡物品的识别率，且更适合在弱算力边缘设备上运行。
- **在多目标跟踪(MOT)方面**，从经典的 SORT、DeepSORT，发展至近期无需强特征提取重识别的 ByteTrack 算法，极大地减少了漏检情况，并在实时性上具有显著优势。
- **在智能家居应用方面**，国内研究大多集中于人体行为分析（跌倒检测、安防入侵）和静态场景识别。将 AI 目标检测深度结合结构化数据库检索，用于“家庭物品数字空间资产化及轨迹寻回”的开源工程与文献相对较少；另外，如何在浏览器端解决视频硬件推流到云端进行低延迟推理（WebRTC/WebSocket技术），也是当前云边协同研究的热点。

## 3. 主要研究内容
本项目主要研究并实现在 Web 平台上驱动的智能家居视觉感知与检索系统，具体研究内容包括：
1. **室内核心物品数据集的构建与处理**：对 MS COCO 2017 数据集进行二次清洗与提取，构建专属于室内智能家居的 `SmartHome-COCO` 数据集（包含20类家具与高频小物品）。
2. **轻量化实时目标检测模型的应用与调优**：基于最新的 YOLOv11（或 YOLOv8）架构进行领域训练验证，在保证 FPS > 30 的同时提升对局部遮挡物品的识别精度。
3. **多目标跟踪与轨迹数据清洗算法**：引入 ByteTrack 追踪算法，解决由于摄像头噪点和光照突变引起的“目标闪烁”问题，并设计内存聚合算法过滤无效的高频数据库写入，实现持久化落库。
4. **流媒体双向通信机制研究**：研究并实现云端拉流（RTSP、本地重播）与客户端流媒体推流（基于浏览器 `getUserMedia` 采集和 WebSocket 推流的云端推理架构），彻底打通跨网络环境的摄像头调用限制。
5. **前后端一体化的大屏系统研发**：后端基于 FastAPI 和 SQLAlchemy 构建高并发响应接口；前端采用 Vue3 + Element Plus 构建具备实时渲染 Bbox、数据图表统计及空间历史截帧检索能力的数据可视化看板。

## 4. 拟采用的研究思路（方法、技术路线、可行性论证等）
**研究方法与技术路线：**
1.  **数据层**：编写 Python 脚本解析 COCO 标注文件，剥离无需的室外类别，输出 YOLO Txt 格式语料用于模型训练。持久化数据采用 MySQL 8.0。
2.  **算法层**：选用 PyTorch 作为底层张量计算框架，利用 Ultralytics 库进行 YOLO 核心推理调用；帧率稳定性依靠 Python 多线程/队列机制进行解耦处理。
3.  **应用层**：
    *   **后端**：采用 FastAPI，基于 ASGI 协议应对 WebSocket 流媒体的高频收发请求。利用 SQLAlchemy 的 ORM 映射管理各类物品的历史发现轨迹。
    *   **前端**：基于 Vite + Vue3 搭建，结合 Canvas 结合二进制图片数据在 Web 端实时绘制检测框（Bbox）；结合 Echarts 实现统计数据的动态呈现。
4.  **部署层**：编写完整 Dockerfile 与 docker-compose 进行全容器化包装及 Nginx 跨网反向代理。

**可行性论证：**
*   **技术可行性**：YOLO 系列及 FastAPI 均为业界成熟且活跃的开源框架，课题应用有充分的社群沉淀支持；硬件方面由于采用云端 GPU 推理 + 浏览器前端推流策略，绕开了客户端硬件算力瓶颈。
*   **工程可行性**：系统已规划为前后端分离架构，组件耦合度极低，单节点瓶颈可通过 Docker 分布式拆分解决。

## 5. 研究工作安排及进度
1. **第 1 - 2 周**：查阅文献资料，撰写并修改开题报告，搭建基础开发环境。
2. **第 3 - 4 周**：编写 `data_pipeline` 脚本提取 SmartHome-COCO 数据集，完成 YOLO 模型的初版微调与评估验证。
3. **第 5 - 6 周**：设计并建立 MySQL 业务数据库模型，跑通 FastAPI 与 YOLO 引擎的基础连通与视频拉流（RTSP/系统内视频文件）。
4. **第 7 - 8 周**：攻克后端连续追踪去重算法（ByteTrack融合）与持久化数据库写入，消除死锁问题。
5. **第 9 - 10 周**：开发 Vue 3 前端数据可视化大屏（监控追踪、日志检索、历史查询功能），并独创实现基于浏览器的 WebRTC/WebSocket 客户端推流模式。
6. **第 11 - 12 周**：系统联合调试与整体 Docker 容器化重构，对前端 Nginx 路由及后端通信进行公网适配。
7. **第 13 - 14 周**：撰写毕业设计论文初稿，制作答辩 PPT 及系统演示录屏。
8. **第 15 - 16 周**：根据指导教师意见修改定稿，参加最终答辩。
